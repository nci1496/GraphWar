# GraphWar

GraphWar 是一个基于 `pygame-ce` 的古代城池攻防原型，核心是“前线作战 + 后勤运输 + 据点治理”。

## 运行

```bash
pip install -r requirements.txt
python main.py
```

## 主要玩法

- 地图由都城、城镇、村庄、关隘和道路构成，支持小/中/大地图。
- 派兵会拆分为多支兵团分批行军；相向兵团会在道路上触发遭遇战。
- 资源是据点本地库存（粮草/金钱），需要通过运输队沿道路转运。
- 出征与驻军都会耗粮；断粮会导致城防归零并提升动荡风险。

## 发展线与防御规则（当前版本）

- 基础据点：
  - 村庄：初始防御 `0`，上限 `1`，屯兵上限 `15`
  - 城镇：初始防御 `1`，上限 `2`，屯兵上限 `25`
- 经济线（高产粮/高产金）：
  - 防御上限：`2 -> 2 -> 3`
  - 屯兵上限：`30 -> 35 -> 40`
- 军事线（高防御/快征兵）：
  - 防御上限：`3 -> 5 -> 7`
  - 屯兵上限：`50 -> 70 -> 120`
  - 升级后不再自产粮，依赖后方运粮

## 屯兵与粮耗

- 驻军基础耗粮较高（中高压）。
- 军事线升级后驻军耗粮额外放大（高压测试设定）。
- 超出屯兵上限的兵力按 **10 倍耗粮** 计算，便于“围城断粮”打法生效。

## AI（规则评分版）

- AI 已独立到 `graphwar/ai_logic.py`。
- 决策目标：`argmax(下一步动作后得分 - 当前得分)`。
- 候选动作：进攻、增援、运粮、运钱、修路、加固、经济/军事升级。
- 升级策略会综合：
  - 交通要道/枢纽、关隘、距离敌前线
  - 粮食承压与军经配比（避免全图军事线导致断粮）

## 起义系统（新增）

- 支持 `100%` 出兵（按钮与快捷键 `4`）。
- 若 100% 出兵导致空城，且民心低于 60，则高概率爆发起义（绿色势力）。
- 起义可连锁扩散到周边“低粮或低民心”据点。
- 起义军会独立扩张和作战；一旦与敌军接触，会并入敌军（转红）。

## 操作

- 左键选择我方据点，再点目标执行行动。
- `1/2/3/4`：出兵比例 `35% / 50% / 75% / 100%`
- `R`：重开一局
- `Esc`：取消当前选择
- `F2`：开发者模式（点据点加兵和加人口）
